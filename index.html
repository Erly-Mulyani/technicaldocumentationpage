Instalasi

Dengan asumsi Anda telah menginstal Node.js, buat direktori untuk menampung aplikasi Anda, dan jadikan itu sebagai direktori kerja Anda.

        $ mkdir myapp
$ cd myapp

Gunakan perintah npm init untuk membuat file package.json untuk aplikasi Anda. Untuk informasi lebih lanjut tentang cara kerja package.json, lihat Spesifikasi penanganan package.json npm.

        $ npm init

Perintah ini meminta Anda untuk beberapa hal, seperti nama dan versi aplikasi Anda. Untuk saat ini, Anda cukup menekan RETURN untuk menerima default untuk kebanyakan dari mereka, dengan pengecualian berikut:

        entry point: (index.js)

Masukkan app.js, atau apa pun yang Anda inginkan untuk nama file utama. Jika Anda menginginkannya menjadi index.js, tekan RETURN untuk menerima nama file default yang disarankan.

Sekarang instal Express di direktori myapp dan simpan di daftar dependensi. Sebagai contoh:

$ npm install express --save

Untuk menginstal Express sementara dan tidak menambahkannya ke daftar dependensi:

$ npm install express --no-save
Hello World
       const express = require('express')
const app = express()
const port = 3000

app.get('/', (req, res) => res.send('Hello World!'))

app.listen(port, () => console.log(`Example app listening at http://localhost:${port}`))

Aplikasi ini memulai server dan mendengarkan pada port 3000 untuk koneksi. Aplikasi tersebut merespons dengan “Hello World!” untuk permintaan ke root URL (/) atau rute. Untuk setiap jalur lainnya, ini akan merespons dengan 404 Not Found.

Contoh di atas sebenarnya adalah server yang berfungsi: Lanjutkan dan klik URL yang ditampilkan. Anda akan mendapatkan tanggapan, dengan log waktu nyata di halaman, dan setiap perubahan yang Anda buat akan tercermin dalam waktu nyata. Ini didukung oleh RunKit, yang menyediakan taman bermain JavaScript interaktif yang terhubung ke lingkungan Node lengkap yang berjalan di browser web Anda. Di bawah ini adalah petunjuk untuk menjalankan aplikasi yang sama di komputer lokal Anda.

Running Locally

Pertama buat direktori bernama myapp, ubah dan jalankan npm init. Kemudian instal express sebagai dependensi, sesuai panduan penginstalan.

Di direktori myapp, buat file bernama app.js dan salin kode dari contoh di atas.

Jalankan aplikasi dengan perintah berikut:

        $ node app.js

Kemudian, muat http: // localhost: 3000 / di browser untuk melihat hasilnya.

Express generator

Gunakan alat pembuat aplikasi, generator-ekspres, untuk membuat kerangka aplikasi dengan cepat.

Anda dapat menjalankan generator aplikasi dengan perintah npx (tersedia di Node.js 8.2.0).

        $ npx express-generator

Untuk versi Node sebelumnya, instal generator aplikasi sebagai paket npm global, lalu luncurkan.

        $ npm install -g express-generator
$ express

Dasat Routing

Perutean mengacu pada menentukan bagaimana aplikasi merespons permintaan klien ke titik akhir tertentu, yang merupakan URI (atau jalur) dan metode permintaan HTTP tertentu (GET, POST, dan sebagainya).

Setiap rute dapat memiliki satu atau lebih fungsi penangan, yang dijalankan ketika rute cocok.

Definisi rute mengambil struktur berikut:

        app.METHOD(PATH, HANDLER)

Dimana:

  • aplikasi adalah turunan ekspres.
  • METHOD adalah metode permintaan HTTP, dalam huruf kecil.
  • PATH adalah jalur di server.
  • HANDLER adalah fungsi yang dijalankan ketika rute cocok.
  • Dan sebagainya.

Contoh berikut menggambarkan definisi rute sederhana.

Tanggapi dengan Hello World! di beranda:

        app.get('/', function (req, res) {
  res.send('Hello World!')
})

Menanggapi permintaan POST di root route (/), beranda aplikasi:

        app.post('/', function (req, res) {
  res.send('Got a POST request')
})

Menanggapi permintaan PUT ke / pengguna rute:

        app.put('/user', function (req, res) {
  res.send('Got a PUT request at /user')
})

Menanggapi permintaan HAPUS ke / pengguna rute:

        app.delete('/user', function (req, res) {
  res.send('Got a DELETE request at /user')
})
File Statis

Untuk menyajikan file statis seperti gambar, file CSS, dan file JavaScript, gunakan fungsi middleware bawaan express.static di Express.

Tanda tangan fungsinya adalah:

        express.static(root, [options])

Argumen root menentukan direktori root untuk menyajikan aset statis. Untuk informasi lebih lanjut tentang argumen opsi, lihat express.static.

Misalnya, gunakan kode berikut untuk menyajikan gambar, file CSS, dan file JavaScript di direktori bernama publik:

        app.use(express.static('public'))

Sekarang, Anda dapat memuat file yang ada di direktori publik:

        http://localhost:3000/images/kitten.jpg
http://localhost:3000/css/style.css
http://localhost:3000/js/app.js
http://localhost:3000/images/bg.png
http://localhost:3000/hello.html
app.use(express.static('public')) app.use(express.static('files'))

Express mencari file dalam urutan yang Anda setel direktori statis dengan fungsi middleware express.static.

Untuk membuat awalan jalur virtual (di mana jalur tersebut sebenarnya tidak ada dalam sistem file) untuk file yang dilayani oleh fungsi express.static, tentukan jalur pemasangan untuk direktori statis, seperti yang ditunjukkan di bawah ini:

        app.use('/static', express.static('public'))

Sekarang, Anda dapat memuat file yang ada di direktori publik dari awalan / jalur statis.

        http://localhost:3000/static/images/kitten.jpg
http://localhost:3000/static/css/style.css
http://localhost:3000/static/js/app.js
http://localhost:3000/static/images/bg.png
http://localhost:3000/static/hello.html

Namun, jalur yang Anda berikan ke fungsi express.static relatif terhadap direktori tempat Anda meluncurkan proses node. Jika Anda menjalankan aplikasi ekspres dari direktori lain, lebih aman menggunakan jalur absolut dari direktori yang ingin Anda layani:

        app.use('/static', express.static(path.join(__dirname, 'public')))

Untuk detail selengkapnya tentang fungsi serve-static dan opsinya, lihat serve-static.
